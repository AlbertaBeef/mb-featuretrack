# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`mb-featuretrack` — a tool to annotate video features with descriptive, tracking overlays (see `README.md`).

## Run

```bash
python3 mb-featuretrack.py <video-path>                 # edit mode (default): annotate + track
python3 mb-featuretrack.py <video-path> --mode render   # render mode: export annotated video
# e.g. python3 mb-featuretrack.py epyc_server_bringup_02_final_02.mp4
```

Two **major modes** (`--mode`): `edit` (default) is the full authoring UI. `render` runs **headless** and burns the saved overlays into a new video **`<video-without-ext>.annotated.mp4`** — full resolution, with the **source audio** muxed back in via ffmpeg (falls back to a video-only OpenCV writer if ffmpeg is absent). It reads the saved JSON; no window, no editing.

Requires OpenCV (`cv2`). Flags: `--json PATH` (default: video path with its extension replaced by `.annotations.json`), `--box N` tracker seed size, `--radius N` default circle radius, `--color R G B` default annotation color (RGB, default white), `--font-scale F` default text size (default 1.2), `--max-width N` display downscale (coords stay full-res), `--smooth N` circle-steadying window (default 2, `0`=off), `--reveal N` reveal-animation length in frames (default 12, `0`=instant), `--mode {edit,render}`, `--font PATH` TTF for label text (default: bundled `fonts/VarelaRound-Regular.ttf`).

**Per-annotation display specs** live in the JSON (editable per feature): `color` (`[R,G,B]`, default white — circle/line/**box**), `text_color` (`[R,G,B]`, default teal `0x64A19D` — text only), `radius` (px), `font_scale`, `line_side` (`"bottom"` default / `"top"` / `"left"` / `"right"` — which box edge the connector attaches to; bottom = label above the feature, top = below, left = to the right of, right = to the left of), `show_text` (off → circle only), `show_underline`, `show_box` (rounded box behind text). All default on/present-value via `ann.get(...)`. New annotations inherit the `--color`/`--text-color`/`--radius`/`--font-scale`/`--line-side` defaults; these flags only seed new annotations — they don't override values already in the JSON.

- **Navigation:** `SPACE`/`p` pause, `d`/→ step forward, `a`/← step back, drag the **Frame** slider to seek, `q`/`ESC` quit (auto-saves).
- **Annotation:** press **`f`** to arm feature placement (`f` again cancels), then left-drag from a feature to the text position, then type the label in-window (`Enter` confirm, `Esc` cancel, `Backspace` edit). **Type an annotation's id** (shown next to each circle) then: `e` stop its track at the current frame (`e` again clears; `r` clears from any frame), `t` toggle its text label on/off, `l` toggle its underline on/off. `u` undo last, `s` save now. (color/radius/font are edited in the JSON per feature.)

## Code

**`mb-featuretrack.py`** — the only source file: a video navigator with mouse-driven, tracked feature annotation. Structured as a single `App` class.

- **Modes** (`--mode` → `self.render` / `self.show_chrome`): `edit` is the full authoring UI. `render` short-circuits `run()` into `export()` (headless), stripping all chrome and editing. Beware the naming: `self.render` is the *mode boolean*; the per-frame draw method is `show_frame()` (renamed to avoid shadowing the attribute). `self.show_chrome == not self.render` gates trackbar creation/sync, `draw_pending`, and `draw_status`.
- **Export** (`export` + `_export_ffmpeg`/`_export_opencv` + `compose_frame`/`_composed_frames`): render mode writes `<video-without-ext>.annotated.mp4`. `compose_frame()` returns the full-res overlaid frame (shared with `show_frame()`, which adds the display downscale). Preferred path pipes raw `bgr24` frames to **ffmpeg** stdin (`libx264` + `-map 1:a:0?` to copy the source audio, optional so silent videos still work); no-ffmpeg fallback is a `cv2.VideoWriter` (mp4v, **no audio**). Verified end-to-end: 1021-frame 1080p sample re-encoded with audio + teal overlay burned in.
- **Navigation loop** (`App.run`) follows the `mb-opencv-navigation` skill's pattern exactly — explicit `frame_idx` (never `CAP_PROP_POS_FRAMES` for UI state), trackbar read *after* `waitKey`, independent `if`/`if` for seek-vs-playback, drag never auto-pauses, 30ms/frame-rate-paced waits. **Preserve these invariants when editing the loop** — each guards a specific navigation bug documented in the skill. Seeking goes through `self.seek_target` (set by the trackbar in edit mode, or by `a`/`d` keys in either mode) so it works with **no trackbar** in render mode. Navigation is skipped while `mode == MODE_TEXT` (typing a label stays on the current frame).
- **Annotation authoring**: gated by `self.armed` — `on_mouse` ignores button-down unless the user pressed **`f`** first, so stray clicks can't create features (`f` toggles; `cancel_pending` clears it, so it's one annotation per `f`). Once armed, `on_mouse` implements click-feature → drag-text → release. Release enters `MODE_TEXT` (in-window typing, no terminal prompt); `finalize_annotation` seeds a tracker box around the click and commits the annotation. Coords are mapped display→full-res via `display_scale`, so all geometry is stored in full-resolution frame space.
- **Tracking** (`advance_trackers`): each annotation owns an OpenCV tracker (`make_tracker`: CSRT→KCF→MIL fallback). **High-water-mark rule** — a tracker only advances when the new frame is exactly `_tracker_frame + 1`, so scrubbing/seeking never feeds it non-consecutive frames. Loaded annotations lazily re-acquire a tracker at their last recorded frame to allow extension. Lost tracking freezes at the last position (drawn gray).
- **Display smoothing** (`display_position`): CSRT box centers wobble frame-to-frame; the drawn position is the mean of recorded positions over a `+/-smooth` frame window. **Render-time only** — the stored `track` keeps raw tracker output, so smoothing is non-destructive, tunable, seek-safe, and applies to loaded/render data too. Circle radius is fixed (`self.radius`), independent of the tracker's drifting box size.
- **Reveal animation** (`reveal_progress` + `_seg` + partial-draw args on `draw_connector`/`draw_label`): over `--reveal` frames from `start_frame`, an annotation draws in stages — circle, then the line grows toward the underline midpoint, then the underline grows from its center, then the text types in. Frame-based, so it's deterministic and replays in render mode. Phase boundaries are the `R_LINE`/`R_UNDER`/`R_TEXT` module constants.
- **Stop frame** (`end_frame` + `sel_buffer` + `set_end_selected`/`clear_end_selected`): each annotation has an optional `end_frame`; past it the overlay is hidden (in render mode entirely; while editing a dim id tag remains so it stays selectable). Selection is keyboard-based — digits build `sel_buffer` (an id) via `select_digit`, which appends only while the result is still a prefix of a real id and otherwise restarts (so with ids 1-5, `1` then `5` selects 5, not 15, while an id like 12 stays typeable); `e` sets/toggles the stop at the current frame, `r` clears it. Right-click was intentionally avoided (WM grabs it on Linux). Ids are drawn beside each circle only when `not self.render` (`draw_id_tag`). Persisted in JSON; `ann.get("end_frame")` is used everywhere so older files without the key still load.
- **Per-annotation specs** (`color`/`radius`/`font_scale`/`show_text`): read at draw time via `ann.get(key, DEFAULT)` so older JSON without them still renders. `color` is stored `[R,G,B]` and converted with the module-level `_bgr()` helper (OpenCV is BGR). `show_text` off draws the circle only (via `draw_connector(..., frac=0)`, skipping the label); `show_underline` off drops the underline; `show_box` draws a filled **rounded** rectangle (in `color`) behind the text, its bottom-center at the anchor so the connector meets it — rounded corners composed in `_rounded_rect` (center rects + AA corner circles, no OpenCV primitive). Box padding uses **fixed font metrics** (`_font_metrics` → `_text_extent(METRIC_REF)`, cached per size), not the label's own glyphs: the box height is constant, the top and bottom borders are equal (`margin = desc + pad_y` above the cap-line and below the baseline), and the baseline stays put — descenders (`p`/`q`/`(`) hang into the bottom margin instead of shifting the text upward. (`_text_extent` renders a string once and measures its pixel ascent/descent; `METRIC_REF` is a fixed caps+ascenders+descenders string.) Text draws in `text_color` on top. The `t`/`l`/`b` keys (`toggle_spec_selected`) flip show_text/show_underline/show_box for the selected annotation; `c` (`flip_line_side_selected`) cycles `line_side` through `LINE_SIDES` (bottom→top→left→right); `draw_label` places the box so the fixed anchor is centered on the chosen edge (top/bottom center the box on the anchor's x, left/right center it on the anchor's y). `color`/`text_color` are edited in the JSON. Defaults come from `--color`/`--radius`/`--font-scale` → `self.default_color`/`self.radius`/`self.font_scale`, baked into new annotations by `finalize_annotation`.
- **Label font** (`_load_font`/`text_size`/`_put_text` + `--font`): label text renders through **`cv2.freetype`** with a real TTF (default: bundled **Varela Round** in `fonts/`) — `cv2.putText` can't load TTFs. `self.ft` is the FreeType2 handle or `None`; when `None` (no font / no `cv2.freetype` / load error) it **falls back to the Hershey font** automatically. Both text drawing and measurement go through `_put_text`/`text_size` so centering, underline width and the typewriter reveal stay correct in either mode. `font_scale` is a multiplier → pixel height `font_scale * BASE_FONT_PX` (32) for the TTF; Hershey still uses it as its native scale. Chrome text (status bar, id tags) stays Hershey. The `fonts/` dir ships `VarelaRound-Regular.ttf` + its `OFL.txt` license.
- **Fixed text anchor** (`text_anchor` + `App.text_anchor()`): the label is anchored at an **absolute** frame position (`text_anchor = [x, y]`), so it stays put while the circle tracks the feature and the connecting line stretches between them — the text does NOT move with the feature. Legacy files storing a feature-relative `text_offset` are migrated to a fixed `text_anchor` on `load()` (computed against the seed feature position) and drop `text_offset` on next save.
- **Persistence** (`load`/`save`): sidecar JSON holds each annotation's seed (`start_frame`, `seed_bbox`, `text_anchor`, `text`, plus the display specs above) **and** its recorded per-frame `track` (`{frame: [cx,cy,w,h]}`). `position_at` renders from the recorded track (exact frame, else hold-last), so render mode is deterministic and fully seekable with no tracker. Runtime keys are prefixed `_` and stripped on save. Auto-saves on quit when dirty.

**Testing note:** core logic and the real-video tracker path are verified headlessly, but the interactive GUI (mouse/keyboard in a live window) can't be exercised in a headless environment — smoke-test window interaction manually when changing the loop or mouse handling.

There is still **no build system, dependency manifest, or automated test suite.** Do not assume build/lint/test commands exist. When adding dependency/test tooling (e.g. `requirements.txt`/`pyproject.toml`, a test runner), document the commands here.

Python + OpenCV (`cv2`, 4.6.0 confirmed in the environment).
